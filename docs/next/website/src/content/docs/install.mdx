---
title: Install Herdr
description: Build, install, upgrade, and verify this Herdr fork from source.
---

This fork is installed from source on Linux or macOS x86_64/aarch64. It does not publish or consume Herdr release channels, download release binaries, check for product announcements, or update itself.

## Install from source

Clone `peisongxiao/herdr`, then use the local installer from that checkout:

```bash
git clone https://github.com/peisongxiao/herdr
cd herdr
./install-local.sh --check
./install-local.sh
./install-local.sh --clean-install
```

The script supports Linux and macOS on x86_64 and aarch64. It requires Bash, usable `cargo` and `rustc` commands, exactly Zig `0.15.2`, and a working C compiler/linker. It uses `CC` when set; otherwise it selects the first available command from `cc`, `clang`, and `gcc`. It reports each missing or unusable build tool explicitly but does not run a package manager for you. Set `ZIG=/path/to/zig` when the required Zig is not the default `zig` on `PATH`. Cross-target Cargo builds are rejected because the installer only installs a native binary from the checkout.

`--check` runs `cargo --version` and `rustc --version`, compiles and links a minimal C program with the selected compiler, validates the exact Zig version, and reports the platform, resolved Zig executable, isolated build cache, and destination without building Herdr. The default is a locked release build installed atomically to `$HOME/.local/bin/herdr`. Build artifacts use a checkout-specific directory under `${XDG_CACHE_HOME:-$HOME/.cache}/herdr/install-local`, preventing artifacts produced from another checkout path or a Docker bind mount from contributing stale native-library paths. An explicit `CARGO_TARGET_DIR` overrides that cache. Use `--prefix DIR` (or `PREFIX`) to change the install prefix, `--bin-dir DIR` to choose the binary directory directly, and `--debug` for a debug build.

After the binary is atomically installed, the script removes obsolete updater data for the selected release or debug profile: `release-notes.json` from the config directory, plus `product-announcements.json` and the old `agent-detection` cache from the state directory. A failed build or binary replacement leaves those files untouched. Local detection overrides under the config directory's `agent-detection/` remain intact.

`--clean-install` is an explicitly destructive factory reset for the selected profile: release removes `herdr`, while `--debug` removes `herdr-dev`. The flag itself authorizes the reset. It stages the candidate first, stops detected default and named-session servers, quarantines and removes that profile's configuration/state trees including all sessions and remote recovery tickets, then installs the candidate. It refuses to run inside Herdr because stopping the owning server would interrupt installation. Deleting recovery tickets can strand hidden remote processes. External agent/MCP/SSH configuration, projects, the opposite profile, and build caches are not removed. The script warns when the destination is not on `PATH`; direct SSH federation still discovers the default `~/.local/bin/herdr` location.

Agent integrations are separate and are not installed or upgraded by `install-local.sh`.

## Verify

Start Herdr:

```bash
herdr
```

If your shell cannot find `herdr`, restart the terminal or check that the install directory is on your PATH.

## Upgrade from source

Pull the desired revision in the checkout and rerun the installer:

```bash
git pull --ff-only
./install-local.sh
```

There is no `herdr update` command or stable/preview channel in this fork. Replacing the binary does not replace an already-running server process. Stop and restart the default or named session when you are ready for it to use the new build.

## Requirements

The source installer supports Linux and macOS on x86_64 and aarch64.
