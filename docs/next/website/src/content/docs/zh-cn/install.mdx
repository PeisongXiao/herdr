这个 fork 仅通过源码安装,支持 x86_64/aarch64 的 Linux 和 macOS。它不发布或读取 Herdr 发布通道,不下载发布二进制,不检查产品公告,也不会自行更新。

## 从源码安装

克隆 `peisongxiao/herdr`,然后在 checkout 中运行本地安装器:

```bash
git clone https://github.com/peisongxiao/herdr
cd herdr
./install-local.sh --check
./install-local.sh
```

仅在确认要完全重置时使用显式破坏性的模式:

```bash
./install-local.sh --clean-install
```

脚本支持 x86_64 和 aarch64 的 Linux 与 macOS。它需要 Bash、可用的 `cargo` 与 `rustc`、严格为 `0.15.2` 的 Zig,以及可工作的 C 编译器/链接器。设置了 `CC` 时使用它,否则依次寻找 `cc`、`clang`、`gcc`。脚本会分别报告缺失或不可用的工具,但不会调用包管理器。所需 Zig 不在默认 `PATH` 时可设置 `ZIG=/path/to/zig`。安装器只安装当前 checkout 的原生二进制,因此拒绝 Cargo 跨目标构建。

`--check` 会在不构建 Herdr 的情况下检查依赖并报告平台、Zig 路径、隔离的构建缓存和安装目标。默认执行锁定的 release 构建,再原子替换 `$HOME/.local/bin/herdr`。构建产物按 checkout 路径隔离在 `${XDG_CACHE_HOME:-$HOME/.cache}/herdr/install-local` 下;显式 `CARGO_TARGET_DIR` 会覆盖它。用 `--prefix DIR` 或 `PREFIX` 改安装前缀,用 `--bin-dir DIR` 指定二进制目录,用 `--debug` 选择 `herdr-dev` profile。

二进制原子安装成功后,脚本会删除所选 profile 的旧 updater 数据:config 目录中的 `release-notes.json`,以及 state 目录中的 `product-announcements.json` 和旧 `agent-detection` 缓存。构建或二进制替换失败时会保留这些文件。config 目录中 `agent-detection/` 下的本地检测覆盖也会保留。

`--clean-install` 是只针对所选 profile 的 factory reset。它先 stage candidate,停止检测到的 default 与 named session server,再 quarantine 并删除该 profile 的 config/state tree、全部 session 和 remote recovery ticket。脚本拒绝从 Herdr 内运行。删除 recovery ticket 可能留下 hidden remote process。外部 agent/MCP/SSH 配置、project、另一 profile 与 build cache 都不会被删除。

智能体集成独立管理,`install-local.sh` 不会安装或升级它们。

## 验证

```bash
herdr
```

如果 shell 找不到 `herdr`,重启终端,或检查安装目录是否在 `PATH` 上。

## 从源码升级

在 checkout 中取得所需 revision,然后重新运行安装器:

```bash
git pull --ff-only
./install-local.sh
```

这个 fork 没有 `herdr update` 命令或 stable/preview 通道。替换二进制不会替换已经运行的 server process。准备好使用新 build 时,停止并重新启动 default 或 named session。

## 系统要求

源码安装器支持 x86_64 和 aarch64 的 Linux/macOS。
