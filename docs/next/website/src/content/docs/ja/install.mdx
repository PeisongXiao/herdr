この fork は Linux または macOS の x86_64/aarch64 でソースからインストールします。Herdr の release channel を公開または参照せず、release binary の download、product announcement の確認、自己 update は行いません。

## ソースからインストール

`peisongxiao/herdr` を clone し、その checkout からローカルインストーラーを使います:

```bash
git clone https://github.com/peisongxiao/herdr
cd herdr
./install-local.sh --check
./install-local.sh
```

必要な build tool を確認した後で完全に reset したい場合だけ、明示的に destructive な mode を使います:

```bash
./install-local.sh --clean-install
```

script は x86_64/aarch64 の Linux と macOS に対応します。Bash、利用可能な `cargo` と `rustc`、正確に Zig `0.15.2`、動作する C compiler/linker が必要です。`CC` が設定されていればそれを使い、なければ `cc`、`clang`、`gcc` の順で探します。不足または利用不能な tool は個別に報告しますが、package manager は実行しません。必要な Zig が default の `PATH` にない場合は `ZIG=/path/to/zig` を設定します。checkout の native binary だけを install するため、Cargo の cross-target build は拒否されます。

`--check` は依存関係、platform、解決した Zig、隔離 build cache、install destination を build なしで表示します。default は locked release build を原子的に `$HOME/.local/bin/herdr` へ install します。artifact は `${XDG_CACHE_HOME:-$HOME/.cache}/herdr/install-local` 配下で checkout path ごとに隔離されます。明示的な `CARGO_TARGET_DIR` はこれを上書きします。`--prefix DIR` または `PREFIX` で prefix、`--bin-dir DIR` で binary directory、`--debug` で `herdr-dev` profile を選べます。

binary の原子的な install が成功した後、選択した profile から古い updater data を削除します。config directory の `release-notes.json`、state directory の `product-announcements.json` と古い `agent-detection` cache が対象です。build または binary replacement に失敗した場合、これらの file は残ります。config directory の `agent-detection/` にある local detection override は保持されます。

`--clean-install` は選択した profile 専用の factory reset です。candidate を先に stage し、検出した default/named session server を停止してから、その profile の config/state tree、全 session、remote recovery ticket を quarantine・削除します。Herdr 内からの実行は拒否されます。recovery ticket の削除は hidden remote process を取り残す可能性があります。外部 agent/MCP/SSH 設定、project、反対側の profile、build cache は削除しません。

agent integration は別管理で、`install-local.sh` は install/update しません。

## 動作確認

```bash
herdr
```

shell が `herdr` を見つけられない場合は terminal を再起動するか、install destination が `PATH` に含まれているか確認してください。

## ソースからアップグレード

checkout で必要な revision を取得し、installer を再実行します:

```bash
git pull --ff-only
./install-local.sh
```

この fork には `herdr update` command や stable/preview channel はありません。binary を置き換えても、すでに動作中の server process は置き換わりません。新しい build を使う準備ができたら default または named session を停止して再起動してください。

## 動作要件

source installer は x86_64 と aarch64 の Linux/macOS をサポートします。
